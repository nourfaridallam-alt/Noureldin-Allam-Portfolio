Modular Function Design

The application uses separate functions for each banking operation, promoting code reusability and maintainability. Each function has a single, well-defined responsibility following the Single Responsibility Principle.

1

Displays current balance with currency formatting

2

3

Processes withdrawals with balance checking

4

main()

Balance Display with Formatting

The show_balance function uses Python's f-string formatting to display currency values with exactly two decimal places, maintaining professional financial presentation standards.

def show_balance(balance):
    print("---------------------------------")
    print(f"Your balance is ${balance: .2f}")
    print("---------------------------------")

Deposit Transaction Handling

The deposit function validates user input, rejecting negative amounts while allowing legitimate deposits. It returns the validated amount to be added to the balance, or 0 if validation fails.

def deposit():
    print("---------------------------------")
    amount = float(input("Enter an amount to be deposited: "))
    print("---------------------------------")
    
    if amount < 0:
        print("---------------------------------")
        print("That's not a valid amount")
        print("---------------------------------")
        return 0
    else:
        return amount

Withdrawal with Balance Validation

The withdraw function implements dual validation: checking for insufficient funds and negative amounts. This prevents overdrafts and invalid transactions, maintaining account integrity.

def withdraw(balance):
    print("---------------------------------")
    amount = float(input("Enter amount to be withdrawn: "))
    print("---------------------------------")
    
    if amount > balance:
        print("---------------------------------")
        print("Insufficient funds")
        print("---------------------------------")
        return 0
    elif amount < 0:
        print("---------------------------------")
        print("Amount must be greater than 0")
        print("---------------------------------")
        return 0
    else:
        return amount

Main Program Loop

The main function orchestrates the entire program using a while loop controlled by an is_running flag. It displays the menu, processes user choices, and updates the balance based on transaction results.

def main():
    balance = 0
    is_running = True
    
    while is_running:
        print("---------------------------------")
        print("   Banking Program   ")
        print("---------------------------------")
        print("1. Show Balance")
        print("2. Deposit")
        print("3. Withdraw")
        print("4. Exit")
        print("---------------------------------")
        
        choice = input("Enter your choice (1-4): ")
        
        if choice == '1':
            show_balance(balance)
        elif choice == '2':
            balance += deposit()
        elif choice == '3':
            balance -= withdraw(balance)
        elif choice == '4':
            is_running = False
            print("Thank you Have a nice day!!")
        else:
            print("That is not a valid option")

Program Flow Architecture

Menu-Driven Design Pattern

The application uses a menu-driven architecture where users select numbered options. This design pattern is common in console applications and provides clear, structured interaction.

Transaction Processing

  1. Display Menu: Show available banking operations
  2. Get User Choice: Accept input from 1-4
  3. Validate Choice: Check if input is valid option
  4. Execute Operation: Call appropriate function
  5. Update Balance: Modify account balance if needed
  6. Loop: Return to menu or exit based on choice

State Management

The balance variable serves as the application's state, persisting across transactions within a single session. Each operation either reads or modifies this state, maintaining consistency throughout the program's lifecycle.

Guard Clause Pattern

Both deposit and withdraw functions use guard clauses (early returns) to handle invalid cases first. This keeps the happy path code clean and reduces nesting, improving readability.

Challenges & Solutions

Challenge 1: Balance Update Logic

Problem: Needed to update balance differently for deposits (add) versus withdrawals (subtract).

Solution: Used compound assignment operators: balance += deposit() for deposits and balance -= withdraw(balance) for withdrawals. Functions return 0 on validation failure, preventing incorrect balance updates.

Challenge 2: Input Validation Complexity

Problem: Multiple validation requirements for different transaction types created complex conditional logic.

Solution: Separated validation logic into individual functions. Deposit checks for negative amounts; withdraw checks for both insufficient funds and negative amounts. Each function handles its own validation independently.

Challenge 3: Consistent UI Formatting

Problem: Maintaining consistent visual presentation across all output messages.

Solution: Used repetitive print statements with dash borders. While this creates some code duplication, it ensures consistent visual styling. Could be improved with a formatting helper function.

Challenge 4: Float Precision for Currency

Problem: Floating-point arithmetic can produce imprecise decimal values unsuitable for currency.

Solution: Used f-string formatting with .2f specifier to always display exactly two decimal places, hiding any floating-point precision issues in display while maintaining calculation accuracy.
