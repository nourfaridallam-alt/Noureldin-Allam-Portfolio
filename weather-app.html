def get_weather(event=None):
    city = city_entry.get().strip()
    
    if not city:
        fade_in_text("Please enter a valid city name.")
        return
    
    url = f"https://api.openweathermap.org/data/2.5/weather?q={city}&appid={API_KEY}&units=metric"
    
    try:
        response = requests.get(url)
        data = response.json()
        
        if response.status_code == 200:
            temp = data["main"]["temp"]
            description = data['weather'][0]['description']
            result = f"{city.title()}\n{temp} ¬∞C | {description}"
            fade_in_text(result)
        else:
            fade_in_text(f"{city} not found")
    except Exception as e:
        fade_in_text("Error fetching city.")

Typing Animation Effect

The fade_in_text function creates a typewriter effect by recursively displaying one character at a time. It uses the after() method to schedule character additions with a 30ms delay, creating smooth visual feedback.

def fade_in_text(text):
    result_label.config(text="", fg="white")
    
    def type_char(i=0):
        if i <= len(text):
            result_label.config(text=text[:i])
            root.after(30, lambda: type_char(i+1))
    type_char()

HTTP Status Code Handling

The application checks the response status code to determine success or failure. A 200 status indicates success, while 404 means the city wasn't found. This enables appropriate user feedback based on the API response.

Challenges & Solutions

Challenge 1: Asynchronous API Requests

Problem: API requests could freeze the GUI while waiting for the server response.

Solution: While this implementation uses synchronous requests, the quick response time and error handling prevent noticeable freezing. For production, could implement threading or async/await patterns for non-blocking requests.

Challenge 2: Custom Button Styling

Problem: Standard Tkinter buttons don't support gradient backgrounds or advanced styling.

Solution: Used ttk.Style() with the "clam" theme to create custom button appearances. Configured foreground, background, font, padding, and relief properties for a modern look.

Challenge 3: Smooth Animation Implementation

Problem: Instant text updates felt abrupt and lacked visual appeal.

Solution: Implemented recursive typing animation using root.after() for scheduled updates. Each character appears with a 30ms delay, creating a smooth typewriter effect.

Challenge 4: JSON Data Navigation

Problem: Extracting nested data from complex JSON response structure.

Solution: Used dictionary key access with bracket notation: data["main"]["temp"] and data['weather'][0]['description']. This directly navigates the nested JSON structure.

Challenge 5: Error Handling for Network Issues

Problem: Network failures or invalid responses could crash the application.

Solution: Wrapped API call in try-except block to catch all exceptions. Provides user-friendly error messages instead of exposing technical errors or crashing.
