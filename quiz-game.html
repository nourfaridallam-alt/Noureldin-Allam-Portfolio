Project Overview

An interactive computer hardware quiz game that tests users' knowledge about common computer components and their acronyms. The game presents five questions about CPU, GPU, RAM, PSU, and includes a fun bonus question, tracking the user's score throughout and providing a final percentage.

This project demonstrates fundamental programming concepts including conditional statements, string manipulation with the lower() method, score tracking with incremental operators, type conversion, and basic user interaction through console input/output.

Technical Implementation

Program Flow

The quiz follows a linear execution pattern with conditional branching at each question. The program begins with a welcome prompt and opt-in check, then proceeds through five sequential questions.

1

Welcome & Opt-In

Display welcome message and check if user wants to play

2

Ask Question

Present question and capture user input

3

Check Answer

Compare answer (case-insensitive) to correct response

4

Update Score

Increment score for correct answer, decrement for wrong

5

Display Results

Show final score as both raw number and percentage

Score Management System

The score tracking system uses a simple integer variable that starts at 0. The += operator adds 1 point for correct answers, while the -= operator (used in question 1) subtracts 1 for incorrect answers.

score = 0

# For correct answers
if answer.lower() == "central processing unit":
    print("Great Job! lets continue :)")
    score += 1
else:
    print(" Whomp Whomp!! YOU SUCK")
    score -= 1

Case-Insensitive Answer Checking

The program uses the .lower() string method to convert user input to lowercase before comparison. This ensures that "CPU", "cpu", and "Cpu" are all treated identically, improving user experience by accepting various input formats.

answer = input(" what does CPU stand for? ")

if answer.lower() == "central processing unit":
    # Answer is correct
    score += 1

Percentage Calculation

The final score is converted to a percentage using division and multiplication. The score is divided by the total number of questions (5), then multiplied by 100. Type conversion with float() ensures decimal precision, and str() converts the result for string concatenation.

print("You got " + str(score) + " questions correct!!")
print("You got " + str(float(score/5) * 100) + "%")

Early Exit Implementation

The program checks if the user wants to play at the start. If they respond with anything other than "yes", the quit() function terminates the program immediately, preventing unnecessary resource usage.

playing = input("Do you want to play? ")

if playing.lower() != "yes":
    quit()
else:
    print(" Okay! let play :)")

Refactored Version with Data Structures

A more maintainable approach would use a dictionary to store questions and answers, with a loop to iterate through them. This eliminates code repetition and makes adding new questions trivial.

questions = {
    "What does CPU stand for?": "central processing unit",
    "What does GPU stand for?": "graphics processing unit",
    "What does RAM stand for?": "random access memory",
    "What does PSU stand for?": "power supply",
    "Is water wet?": "yes"
}

score = 0

for question, correct_answer in questions.items():
    answer = input(question + " ").lower()
    
    if answer == correct_answer:
        print("Great Job! lets continue :)")
        score += 1
    else:
        print("Whomp Whomp!! YOU SUCK")

print(f"You got {score} questions correct!!")
print(f"You got {(score/len(questions)) * 100}%")

Challenges & Solutions

Challenge 1: Case Sensitivity

Problem: Users might type answers in various capitalizations (CPU, cpu, Cpu), causing incorrect rejection of valid answers.

Solution: Applied the .lower() method to all user inputs before comparison: if answer.lower() == "central processing unit". This normalizes all input to lowercase for consistent checking.

Challenge 2: Score Display Formatting

Problem: Needed to combine score integer with text strings and convert score to percentage.

Solution: Used str() to convert the integer score to string for concatenation, and float() to ensure decimal division: str(float(score/5) * 100). This produces accurate percentage calculations.

Challenge 3: Inconsistent Score Tracking

Problem: First question decrements score for wrong answers, but remaining questions don't, creating inconsistent behavior.

Solution: This is a bug in the current implementation. Should either remove score decrement from question 1, or add it to all questions for consistency. Recommended approach: only increment for correct answers, ignore incorrect answers.

Challenge 4: User Experience on Wrong Answers

Problem: The "YOU SUCK" message might be discouraging for some users, especially younger players.

Solution: Consider alternative feedback messages like "Not quite, try the next one!" or make the tone configurable based on player preference (casual/serious/educational modes).
