Technical Implementation

Object-Oriented Architecture

The calculator is built using object-oriented programming principles, encapsulating all functionality within a Calculator class. This design promotes code organization, reusability, and maintainability.

Calculator Class

Attributes:

• root: Main Tk window instance

• MainFrame: Outer frame container

• WidgetFrame: Inner frame for widgets

• lblDisplay: Label widget for display

• input_button: String to store calculation

Methods:

• __init__(): Constructor - initializes GUI

• create_button(): Factory method for buttons

• button_click(): Event handler for all buttons

GUI Layout Design

The interface uses a nested frame structure with Tkinter's grid geometry manager for precise widget placement. The layout consists of a display area at the top and a 5x4 button grid below.

def __init__(self, root):
    self.root = root
    self.root.title("Calculator")
    self.root.geometry("615x690+400+200")
    self.root.configure(bg='cadet blue')
    
    # Create nested frame structure
    self.MainFrame = Frame(self.root, bd=18, width=600,
                          height=670, relief=RIDGE, bg='powder blue')
    self.MainFrame.grid()
    
    self.WidgetFrame = Frame(self.MainFrame, bd=18, width=590,
                            height=660, relief=RIDGE, bg='cadet blue')
    self.WidgetFrame.grid()
    
    # Display label
    self.lblDisplay = Label(self.WidgetFrame, width=30, height=2,
                           bg='white', fg='black',
                           font=('Arial', 20, 'bold'), anchor='e')
    self.lblDisplay.grid(row=0, column=0, columnspan=4,
                        padx=10, pady=10)

Dynamic Button Creation

The create_button method demonstrates the factory pattern, generating buttons with consistent styling but different labels and positions. Lambda functions bind each button to its specific action.

def create_button(self, text, row, column):
    btnWidget = Button(
        self.WidgetFrame,
        text=text,
        width=6,
        height=2,
        bd=4,
        bg='powder blue',
        font=('Arial', 20, 'bold'),
        command=lambda: self.button_click(text)
    )
    btnWidget.grid(row=row, column=column, padx=5, pady=5)

Event Handling Logic

The button_click method serves as the central event handler, processing all button inputs through conditional logic. It manages different operations including backspace, clear, calculation, and sign toggling.

def button_click(self, text):
    if text == "<-":
        self.input_button = self.input_button[:-1]
    elif text == "CE":
        self.input_button = ""
    elif text == "C":
        self.input_button = ""
    elif text == "=":
        try:
            self.input_button = str(eval(self.input_button))
        except:
            self.input_button = "Error"
    elif text == "+/-":
        self.input_button = str(-float(self.input_button))
    else:
        self.input_button += text
    
    self.lblDisplay.config(text=self.input_button)

Expression Evaluation

The calculator uses Python's built-in eval() function to parse and evaluate mathematical expressions. While convenient, this is wrapped in a try-except block to handle invalid inputs gracefully.

Challenges & Solutions

Challenge 1: Button Event Binding

Problem: Each button needed to pass its specific text value to the event handler when clicked.

Solution: Used lambda functions in the command parameter: command=lambda: self.button_click(text). This creates a closure capturing the button's text value for each instance.

Challenge 2: Display Updating

Problem: The display needed to update after every button click to show the current expression.

Solution: Used the Label's config() method to update the text property: self.lblDisplay.config(text=self.input_button), ensuring real-time feedback.

Challenge 3: Sign Toggle Implementation

Problem: The +/- button needed to negate the current number without affecting the entire expression.

Solution: Converted the input to float, negated it, then converted back to string: str(-float(self.input_button)). This preserves the numeric value while changing its sign.

Challenge 4: Error Handling for Invalid Input

Problem: Users could create invalid expressions (like "5++3") that would crash the calculator.

Solution: Wrapped eval() in a try-except block. On error, display "Error" instead of crashing, maintaining user experience continuity.

Challenge 5: Window Sizing and Layout

Problem: Ensuring the calculator maintained proper proportions and all buttons were visible.

Solution: Set fixed geometry with geometry("615x690+400+200"), specifying width, height, and screen position. Used consistent padding values for uniform spacing.

from tkinter import *

class Calculator:

    def __init__(self, root):
        self.root = root
        self.root.title("Calculator")
        self.root.geometry("615x690+400+200")
        self.root.configure(bg='cadet blue')

        self.MainFrame = Frame(self.root, bd=18, width=600,
                              height=670, relief=RIDGE, bg='powder blue')
        self.MainFrame.grid()

        self.WidgetFrame = Frame(
            self.MainFrame, bd=18, width=590, height=660,
            relief=RIDGE, bg='cadet blue')
        self.WidgetFrame.grid()

        self.lblDisplay = Label(self.WidgetFrame, width=30, height=2,
                               bg='white', fg='black',
                               font=('Arial', 20, 'bold'), anchor='e')
        self.lblDisplay.grid(row=0, column=0, columnspan=4,
                            padx=10, pady=10)

        self.input_button = " "

        self.create_button("<-", 1, 0)
        self.create_button("CE", 1, 1)
        self.create_button("C", 1, 2)
        self.create_button("+/-", 1, 3)

        self.create_button("7", 2, 0)
        self.create_button("8", 2, 1)
        self.create_button("9", 2, 2)
        self.create_button("+", 2, 3)

        self.create_button("4", 3, 0)
        self.create_button("5", 3, 1)
        self.create_button("6", 3, 2)
        self.create_button("-", 3, 3)

        self.create_button("1", 4, 0)
        self.create_button("2", 4, 1)
        self.create_button("3", 4, 2)
        self.create_button("*", 4, 3)

        self.create_button("0", 5, 0)
        self.create_button(".", 5, 1)
        self.create_button("=", 5, 2)
        self.create_button("/", 5, 3)

    def create_button(self, text, row, column):
        btnWidget = Button(
            self.WidgetFrame, text=text, width=6, height=2, bd=4,
            bg='powder blue', font=('Arial', 20, 'bold'),
            command=lambda: self.button_click(text))
        btnWidget.grid(row=row, column=column, padx=5, pady=5)

    def button_click(self, text):
        if text == "<-":
            self.input_button = self.input_button[:-1]
        elif text == "CE":
            self.input_button = ""
        elif text == "C":
            self.input_button = ""
        elif text == "=":
            try:
                self.input_button = str(eval(self.input_button))
            except:
                self.input_button = "Error"
        elif text == "+/-":
            self.input_button = str(-float(self.input_button))
        else:
            self.input_button += text
        self.lblDisplay.config(text=self.input_button)

root = Tk()
App = Calculator(root)
root.mainloop()
